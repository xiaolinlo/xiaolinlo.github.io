使用UV5新建STM32工程项目

发表于 2020-05-04 更新于 2020-09-11

本文将介绍如何使用 UV5 新建基于固件库的 STM32 工程项目。参考资料来自：正点原子《STM32F1开发指南（精英版）— 库函数版本》。

正文

与开发51单片机新建项目相比，STM32 新建项目更加复杂。本文新建工程项目所使用的芯片为STM32F103ZET6，所使用的的固件库版本为V3.5，所使用的工程管理格式仅供参考。

具体步骤

1）新建工程目录

新建一个用于存放此工程项目的目录，后面所建立的工程文件都可以放在这个目录下面，这里我们建立一个名字为 Template 的目录。

2）新建子目录

在 Template 目录下新建 3 个子目录，分别命名为：USER、CORE、OBJ、FWLib、HARDWARE

USER：用于存放工程文件；

CORE：用于存放核心文件；

OBJ：用于存放编译过程产生的中间文件，如 .hex 文件；

FWLib：用于存放固件库外设源码；

HARDWARE：用于存放用户开发的外设源码。

3）新建工程文件

  • 点击 UV5 的菜单栏：Project –> New Uvision Project，定位到此前建立的 Template\ USER 目录下，工程命名为 Template，点击保存；

  • 此后会自动弹出芯片选择窗口，找到所需的芯片型号，如 STM32F103ZE，点击 OK；

  • 此后会自动弹出 Manage Run-Time Environment 窗口，用于添加所需组件，可直接点击 Cancel；

完成这一步后在 USER 目录下会自动生成2个目录和2个文件，Template.uvprojx 即工程文件。 Listings 和 Objects 目录用于存放编译过程产生的中间文件，此处可不使用，因为我们要存放在此前建立的 OBJ 目录下。

4）添加核心文件、启动文件与固件库外设源码

  • 定位到固件库 \Libraries\CMSIS\CM3\CoreSupport 目录下，将文件 core_cm3.c 和 core_cm3.h 复制到此前建立的 CORE 目录下；
  • 定位到固件库 \Libraries\CMSIS\CM3\DeviceSupport\ST\STM32F10x\startup\arm 目录下，将文件 startup_stm32f10x_hd.s （演示所使用的的芯片型号为大容量，对不同容量型号应选择不同启动文件）同样复制到 CORE 目录下；
  • 定位到固件库 \Libraries\CMSIS\CM3\DeviceSupport\ST\STM32F10x 目录下，将文件 stm32f10x.h，system_stm32f10x.c，system_stm32f10x.h 复制到此前建立的 USER 目录之下；
  • 定位到固件库 \Libraries\STM32F10x_StdPeriph_Driver 目录下，将目录 src，inc 目录复制到此前建立的 FWLib 目录下；
  • 定位到固件库 \Project\STM32F10x_StdPeriph_Template 目录下，将文件 main.c，stm32f10x_conf.h，stm32f10x_it.c，stm32f10x_it.h 同样复制到 USER 目录下；

5）在 UV5 中添加文件到工程

  • 右键点击 Target1，选择 Manage Components Items，在 Project Targets一栏，将 Target 名字修改为 Template（可改可不改），然后在 Groups 删掉 Source Group1，建立三个 Groups：USER，HARDWARE，FWLIB，CORE。点击 OK。

  • 添加文件到 Groups 中：

    • 方法一：使用上述方法，在 Manage Components Items 窗口 Files 一栏中，将步骤（4）添加的文件添加到对应的Groups 中；
    • 方法二：直接双击工程栏中的 Groups，将步骤（4）添加的文件添加到对应的Groups 中。

注意：

  • 只有 .h 文件不需要在此添加，即启动文件 startup_stm32f10x_hd.s 同样需要到 Groups 的 CORE 中。
  • 对于固件库外设源码，可在需要使用某外设时才添加相应外设源码到 FWLIB 中，以提高编译速度。

*6）设置编译生成文件的生成路径**

  • 点击魔术棒 -> Output -> Select Folder for Object，定位到此前建立的 OBJ 目录下，点击 OK；

  • 在同一页面上的 Create HEX File 选项打上勾。点击 OK。

7）添加头文件路径与定义全局宏定义变量

  • 添加头文件路径：点击魔术棒 -> C/C++ -> Include Paths，新建多个路径，分别定位到此前建立的包含 .h 文件的最后一级目录下。点击 OK。

  • 定义全局宏定义变量：在同一页面上的 Define 一栏写上：STM32F10X_HD, USE_STDPERIPH_DRIVER。点击 OK。

关于 STM32F10X_HD, USE_STDPERIPH_DRIVER 的作用后文会有提及。
